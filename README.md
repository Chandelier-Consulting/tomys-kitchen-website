## Firebase manager dashboard

The customer site has static fallbacks, and `/dashboard` can save live edits to Firebase:

1. `npx firebase-tools login`
2. Create/select a Firebase project, then enable Authentication Email/Password, Firestore, and Storage.
3. Add a web app in Firebase and set these env vars:

```bash
NEXT_PUBLIC_FIREBASE_API_KEY=
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=
NEXT_PUBLIC_FIREBASE_PROJECT_ID=
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=
NEXT_PUBLIC_FIREBASE_APP_ID=
```

4. Create the manager user in Firebase Auth.
5. Deploy rules with `npm run firebase:rules`.
6. Seed starter menu/images/links with `GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json npm run firebase:seed`.

Managers can then sign in at `/dashboard`, replace images, add menu items, and update delivery links.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.
